# Matriz de afirmaciones y límites — S.N.E.-E.C.O.

Esta matriz define qué se puede afirmar responsablemente sobre **E.C.O. — Entérico Codificador Orgánico** y qué afirmaciones deben evitarse.

## Propósito

Proteger el proyecto de interpretaciones exageradas, clínicas, diagnósticas, forenses o relacionadas con conciencia humana. El objetivo es mantener una presentación clara, técnica, educativa y experimental.

## Afirmaciones permitidas

| Tipo de afirmación | Formulación responsable | Evidencia asociada |
|---|---|---|
| Arquitectura experimental | S.N.E.-E.C.O. es un pipeline experimental y educativo bioinspirado. | Documentos del portafolio y reportes generados. |
| Organización de datos | El sistema organiza entradas, barreras, motilidad, defensa, estados y reportes. | `docs/sne-eco-architecture-map.md` |
| Observabilidad | El sistema genera reportes auditables sobre rutas, recurrencia, estado y comparación contra RC1. | `results/sne_eco_observability_dashboard.json` |
| Portafolio técnico | El proyecto puede ser revisado como demostración de arquitectura, documentación, pruebas y UX técnica. | `docs/sne-eco-evidence-matrix.md` |
| Lenguaje bioinspirado | El sistema usa metáforas del sistema entérico para explicar flujos de datos. | `docs/sne-eco-glossary.md` |

## Afirmaciones no permitidas

| Afirmación a evitar | Por qué debe evitarse | Versión responsable |
|---|---|---|
| E.C.O. diagnostica enfermedades. | El sistema no tiene validación clínica. | E.C.O. organiza datos y reportes educativos. |
| E.C.O. detecta conciencia humana. | No modela conciencia humana ni estados mentales reales. | E.C.O. modela estados internos del pipeline. |
| E.C.O. prueba funcionamiento biológico real. | Es una arquitectura bioinspirada, no un modelo biomédico validado. | E.C.O. usa analogías operativas del sistema entérico. |
| E.C.O. sirve para análisis forense. | No fue diseñado ni validado para ese uso. | E.C.O. es una demo técnica y educativa. |
| E.C.O. predice conducta humana. | El pipeline no analiza personas ni realiza inferencias psicológicas. | E.C.O. analiza rutas informacionales internas del sistema. |

## Regla de comunicación pública

Cuando se explique E.C.O., debe presentarse como un sistema experimental de organización, observabilidad y documentación técnica. La metáfora entérica ayuda a explicar el flujo, pero no convierte el sistema en una herramienta médica.

## Relación con la matriz de evidencias

Este documento complementa `docs/sne-eco-evidence-matrix.md`. La matriz de evidencias muestra qué existe y cómo se valida. Esta matriz define cómo se debe hablar del sistema sin exagerar su alcance.

## Límite responsable

No tiene uso clínico, no diagnóstico, no forense y no pretende modelar conciencia humana. Su función es ordenar afirmaciones responsables sobre un pipeline experimental y educativo.

# Guía rápida de evaluación — S.N.E.-E.C.O.

Esta guía permite revisar el proyecto **E.C.O. — Entérico Codificador Orgánico** en pocos minutos, sin necesitar conocimiento avanzado de programación.

## 1. Qué mirar primero

- `README.md`: entrada general del proyecto.
- `docs/sne-eco-public-summary.md`: explicación pública y breve.
- `docs/sne-eco-portfolio-index.md`: índice de portafolio técnico.
- `docs/case-study-sne-eco-neurogastro-pipeline.md`: caso de estudio del pipeline.

## 2. Comando principal de demostración

```bash
make sne-portfolio-demo
```

Este comando ejecuta el pipeline neurogastrocomputacional, revisa el portafolio técnico y corre las pruebas automatizadas.

## 3. Resultado esperado

Un resultado saludable debería mostrar:

```text
S.N.E.-E.C.O. portfolio demo ready
portfolio check: green
tests passed
```

## 4. Cómo interpretar el proyecto

E.C.O. usa una metáfora bioinspirada: el sistema procesa datos como si fueran paquetes que deben avanzar, ser observados, quedar en vigilancia o ser rechazados según señales internas.

La gracia del proyecto no está en prometer inteligencia biológica real, sino en construir una arquitectura explicable, auditable y comunicable.

## 5. Límite responsable

Este proyecto es experimental y educativo. No tiene uso clínico, diagnóstico, terapéutico, forense ni pretende modelar conciencia humana.

# Autonomía Variable en E.C.O.

## Propósito

La autonomía variable es un marco documental para ajustar niveles de independencia operativa dentro de E.C.O. según contexto, trazabilidad, límites responsables y revisión humana.

Este marco no define autonomía humana, conciencia, voluntad ni libre albedrío real. Describe una forma de ordenar decisiones documentales futuras sobre asistencia, recomendación, simulación y gobernanza operativa.

## Relación con IAFA

IAFA (Índice de Agencia Funcional Adaptativa) puede funcionar como señal documental futura para discutir cambios de nivel dentro del marco de autonomía variable.

En este sprint, IAFA no es métrica productiva actual, no activa decisiones automáticas y no habilita independencia operativa real. Cualquier uso posterior requeriría especificación, validación, trazabilidad y revisión humana.

## Niveles de autonomía variable

### Nivel 0: solo documentación / sin acción automática

El sistema registra, ordena o describe criterios. No recomienda acciones operativas ni ejecuta herramientas.

### Nivel 1: asistencia supervisada

El sistema puede ayudar a organizar información o proponer pasos bajo supervisión humana directa. La revisión humana permanece como condición central.

### Nivel 2: recomendación con revisión humana

El sistema puede formular recomendaciones documentales, pero toda decisión requiere revisión humana antes de avanzar.

### Nivel 3: ejecución simulada con gate documental

El sistema puede describir una ejecución simulada o una ruta de validación sintética con gate documental. No ejecuta acciones autónomas reales.

### Nivel 4: autonomía experimental condicionada, solo futura

El sistema podría estudiar autonomía experimental condicionada únicamente como línea futura. Este nivel no queda habilitado por este sprint y requeriría controles, validaciones y aprobación humana explícita.

## Criterios de escalamiento

Escalar a revisión humana o pausar avance ante:

- Baja confianza.
- Conflicto con límites responsables.
- Ruido alto.
- Desviación de reglas.
- Falta de trazabilidad.
- Riesgo de modificar baseline o umbrales.

## No implementación funcional

Este sprint no implementa lógica funcional, sin telemetría real, sin agentes, sin PID, sin fuzzy logic y sin inferencia activa.

El documento solo define un marco conceptual de gobernanza. No cambia scripts, modelos, fuentes externas, baseline, umbrales, entrenamiento ni datos reales.

## Límites responsables

- Sin datos reales.
- Sin entrenamiento.
- Sin modificación de baseline.
- Sin recalibración de umbrales.
- Sin afirmaciones biomédicas aplicadas.
- Sin ejecución autónoma de herramientas.
- Sin control de agentes.
- Sin decisiones clínicas.
- Sin conciencia.
- Sin libre albedrío real.

# E.C.O. operational state example

## Propósito

Este documento muestra un ejemplo concreto de estado operativo sintético usando el schema operacional de E.C.O.
Su función es demostrar cómo una pieza del sistema puede describirse con campos mínimos, límites responsables y validación reproducible.

## Estado representado

- state_id: eco_synthetic_operational_dashboard_v1
- state_kind: observed
- classification: permitido
- status: passed

## Lectura simple

Este ejemplo representa al dashboard operacional sintético como un estado observado.
Toma reportes sintéticos como entrada y declara salidas JSON/Markdown generadas por el sistema.

## Límite responsable

Este ejemplo es documental y sintético.
No usa datos sensibles, no entrena modelos, no modifica baseline, no recalibra umbrales y no realiza afirmaciones biomédicas aplicadas.

# E.C.O. synthetic state timeline

## Propósito

Esta línea de tiempo organiza la evolución operativa de los estados sintéticos de E.C.O.
Su función es mostrar cómo el sistema pasa desde demos sintéticas aisladas hacia reportes, matriz de señales, dashboard e índice de arquitectura.

## Secuencia operativa

1. Estado definido: se establecen criterios de decisión para demos sintéticas.
2. Estado registrado: las demos permitidas quedan descritas en un registro controlado.
3. Estado ejecutable: cada demo sintética genera salidas JSON y Markdown reproducibles.
4. Estado validado: las salidas se revisan mediante contrato sintético.
5. Estado comparado: las demos se contrastan en un reporte común.
6. Estado matricial: las señales se organizan en una matriz sintética.
7. Estado observado: el dashboard operacional resume componentes y límites.
8. Estado indexado: la arquitectura queda navegable desde un índice documental.

## Lectura simple

E.C.O. no solo ejecuta demos sintéticas.
También las registra, valida, compara, organiza, observa y documenta.

## Límite responsable

Esta línea de tiempo es documental y operativa.
No usa datos sensibles, no entrena modelos, no modifica baseline, no recalibra umbrales y no realiza afirmaciones biomédicas aplicadas.

# Guía de demostración — S.N.E.-E.C.O.

Esta guía explica cómo ejecutar una demostración reproducible del pipeline **E.C.O. — Entérico Codificador Orgánico**.

## 1. Comando principal

```bash
make sne-portfolio-demo
```

Este comando ejecuta:

```text
1. Pipeline neurogastrocomputacional.
2. Chequeo de portafolio técnico.
3. Validación completa de tests.
```

## 2. Qué demuestra

La demo muestra que E.C.O. puede:

- generar reportes auditables;
- revisar rutas confundidas;
- auditar recurrencias;
- consolidar observabilidad;
- explicar estados internos en lenguaje claro;
- validar materiales de portafolio;
- confirmar estabilidad mediante pruebas automatizadas.

## 3. Lectura esperada

Un resultado correcto debe incluir algo similar a:

```text
S.N.E.-E.C.O. neurogastro pipeline ready
S.N.E.-E.C.O. portfolio check ready
189 passed
S.N.E.-E.C.O. portfolio demo ready
```

## 4. Interpretación simple

```text
green = no hay confusión estructural.
attention = hay señales internas que ameritan revisión.
```

En términos simples: el sistema no está fallando; está avisando.

## 5. Valor profesional

Esta demo sirve para presentar E.C.O. como un proyecto técnico verificable, modular y documentado.

Demuestra capacidades en:

- arquitectura de sistemas;
- automatización con Makefile;
- validación con pytest;
- UX conversacional aplicada a reportes técnicos;
- software bioinspirado;
- documentación orientada a portafolio.

## 6. Límite responsable

Esta demostración es educativa y experimental. No tiene uso clínico, diagnóstico, terapéutico, forense ni pretende modelar conciencia humana.
